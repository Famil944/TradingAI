# TradingAI

Telegram-бот для анализа рынка, локальной Paper-торговли и торговли на
Binance Futures Demo/Testnet.

## Безопасность

- По умолчанию используется `TRADING_MODE=DEMO`.
- `DEMO` отправляет настоящие ордера в Binance Futures Demo, а не симулирует
  их локально.
- Локальная симуляция находится в модуле `paper`.
- Live-ордера разрешаются только при `TRADING_MODE=LIVE`, наличии live-ключей
  и точном значении
  `LIVE_TRADING_CONFIRMATION=I_UNDERSTAND_LIVE_RISK`.
- Не храните `.env` и API-ключи в Git.

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e .
Copy-Item .env.example .env
```

Заполните `.env`, затем запустите:

```powershell
.\.venv\Scripts\python.exe preflight.py
.\.venv\Scripts\python.exe main.py
```

`preflight.py` не отправляет ордера: он проверяет ключи, соединение, серверное
время, разрешение торговли, One-way Mode и фильтры инструмента.

Для первого теста установите:

```dotenv
TRADING_MODE=DEMO
TRADING_CLOSE_ONLY=false
```

Команды `/demo_long` и `/demo_short` работают только в Demo. Команды
`/trade_long` и `/trade_short` используют текущий режим. `/trading_mode`
показывает активный режим и состояние Live-блокировки.

Режим также можно выбрать через Telegram:

```text
🏠 Меню → ⚙️ Настройки → 🔁 Demo / Live
```

Переключатель проверяет отсутствие открытых позиций, ключи и соединение.
Для LIVE дополнительно показывается подтверждение риска. После успешной
проверки режим и биржевой клиент меняются сразу, без перезапуска.

Live-контур использует Binance Portfolio Margin UM (`/papi`). Для Live
необходимы все три настройки:

```dotenv
TRADING_MODE=LIVE
BINANCE_LIVE_API_KEY=...
BINANCE_LIVE_API_SECRET=...
LIVE_TRADING_CONFIRMATION=I_UNDERSTAND_LIVE_RISK
```

Для аварийной блокировки новых входов установите
`TRADING_CLOSE_ONLY=true`: закрытие
позиций, отмена ордеров и установка защиты при этом остаются доступны.

Аккаунт Binance должен использовать **One-way Mode**. При Hedge Mode или
расхождении системного времени с Binance более чем на пять секунд запуск
будет остановлен до исправления конфигурации.

API-ключ Live должен иметь только `Enable Reading` и
`Enable Portfolio Margin Trading`, а также ограничение по доверенному IP.
Вывод средств, Universal Transfer и займы боту не нужны.

## Настройки в Telegram

Раздел `⚙️ Настройки` позволяет без изменения файлов выбрать риск на сделку,
максимальный размер позиции, минимальный Quality Score, таймфрейм
автосканирования и режим уведомлений. Значения хранятся в SQLite и применяются
к следующему анализу или сделке.

По умолчанию включён консервативный авто-риск: бот уменьшает заданный верхний
лимит риска для менее качественных сигналов и дополнительно ограничивает
позицию 20% доступного баланса. При рестарте Render включённая автоторговля
возобновляется автоматически и продолжает отправлять уведомления в последний
Telegram-чат, из которого она была включена.

## Render

В корне репозитория находится `render.yaml`: он создаёт платный Background
Worker и постоянный диск `/var/data`. В Render выберите **New → Blueprint**,
подключите репозиторий и заполните предложенные секретные переменные.

Перед первым запуском:

1. Оставьте `TRADING_MODE=DEMO`. Blueprint включает Demo-сделки значением
   `TRADING_CLOSE_ONLY=false`; для проверки только чтения временно поставьте
   `true`.
2. Заполните Telegram token и Demo-ключи.
3. После создания сервиса откройте `Connect → Outbound` и добавьте все
   исходящие IP-диапазоны сервиса в whitelist Binance.
4. Остановите локальную копию бота: Telegram polling допускает один активный
   экземпляр с этим токеном.
5. Проверьте баланс, режим One-way и защитные ордера на первой Demo-сделке.

SQLite, состояние автоторговли и выбранный через Telegram режим сохраняются
на диске. Секреты задаются только в Render Environment и не должны попадать
в Git.

## Проверка

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

Перед Live-режимом обязательно проведите длительную проверку на Binance Demo,
сверьте фильтры количества/цены каждого инструмента и убедитесь, что защитные
ордера присутствуют непосредственно на бирже.

Каждая открытая позиция получает биржевые `STOP_MARKET` и
`TAKE_PROFIT_MARKET`. При рестарте бот сверяет позицию, открытые ордера и БД,
восстанавливает отсутствующую защиту и фиксирует закрытия, выполненные биржей.
